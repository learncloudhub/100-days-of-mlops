# Day 2 - Configuring and Running JupyterLab

## Objective

Configure and troubleshoot a JupyterLab server for the xFusionCorp Industries data science team.

## Lab Scenario

The provided JupyterLab configuration contained issues that prevented the server from starting correctly. The goal was to inspect the configuration, fix the problems, and start the server.

## Requirements

- Listen on port **8888**
- Bind to **0.0.0.0**
- Use **/root/notebooks** as the notebook root directory
- Ensure `/root/notebooks` exists

## Solution Steps

1. Activate the Python virtual environment.
2. Create the notebook directory.
3. Update the JupyterLab configuration.
4. Start the JupyterLab server.
5. Verify the server is accessible through the Jupyter UI.

## Outcome

Successfully configured and launched JupyterLab with the required settings.
